## Project Overview

A Tic-Tac-Toe (Tris) game web app built with vanilla HTML, CSS, and JavaScript — no framework, no build system, no dependencies. The UI is in Italian.

## Running the Project

Open `index.html` directly in a browser. There is no build step, no npm, no server required.

## Architecture

All game logic and UI live in three files:

- `index.html` — All screens (menu, shared-screen game, vs-computer game) are present in the DOM simultaneously; JavaScript toggles `display` to navigate between them.
- `tris_javascript.js` — All game logic (~637 lines). Procedural style with global state.
- `tris_css.css` — Styling, animations, and responsive board layout via CSS Grid.

### Game Modes

1. **Schermo Condiviso** — Two humans on the same device.
2. **Gioca Contro il Computer** — Single player vs AI.

Both modes support 3×3 and 4×4 boards.

### State Management

Board state is tracked using two arrays (`controllo_x`, `controllo_o`) that record which cells each player occupies. Win detection (`Verifica_vittoria`) checks all rows, columns, and diagonals against the winning combinations generated by `generaPosizioniVincenti()`.

### AI

The computer opponent (`Minimax()`) is a greedy single-ply strategy: win if possible, block the opponent's win if possible, otherwise pick randomly. It is not a full minimax tree search despite the function name.

### Board Generation

Board cells are created dynamically in JavaScript (`generazione_celle_sc()` for shared-screen, `generazione_celle_gc()` for vs-computer) and injected into the DOM. Each cell gets a click event listener that drives the turn logic.

### Key Functions

| Function | Purpose |
|---|---|
| `Verifica_vittoria()` | Checks win condition against current board state |
| `ApplicaFiltro_Verifica_vittoria()` | Highlights winning cells in yellow |
| `Minimax()` | Selects computer's next move |
| `generaPosizioniVincenti()` | Builds all winning combinations for the chosen board size |
| `Inserimento_x()` | Handles a player's move (places mark, checks win/draw) |
| `ResetPartita()` / `ResetPartita_gc()` | Resets board without clearing scores |
